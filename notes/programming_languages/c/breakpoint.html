Using breakpoints in C

An easy shortcut to get a breakpoint in gdb going is by using global assembly labels.

int fun(int a, int b) {
    // do something

    asm("b:");
    // or
    asm("b: nop");

    // return something
}

The label does nothing, but it means that once gdb is open, you can simply do:

(gdb) b b

Original idea from: - https://nullprogram.com/blog/2024/01/28/

In case the above does not work, please try: - https://github.com/scottt/debugbreak

Using POSIX raise

You can also try:

// main.c
#include "signal.h"

int main() {
  raise(SIGTRAP);
  return 0;
}
// Compiled with gcc -g /tmp/main.c -o /tmp/main && gdb /tmp/main

Then when running the program with gdb, this will effectively pause execution so that you can have a look around. I.e:

Neovim shortcut

  • F1: Adds the breakpoint in a new line under the cursor.
  • F2: Deletes the breakpoint.
vim.api.nvim_create_autocmd({"BufNewFile", "BufRead"}, {
  group = vim.api.nvim_create_augroup("UserCAutoCommands", {clear=true}),
  pattern = '*.c',
  command = [[
    noremap  :normal oasm("b: nop"); |
    noremap  :g/^.*asm("b: nop").*/d
  ]]
})
